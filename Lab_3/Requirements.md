# Lab 03 Requirements

## Thông tin chung

- Môn học: Trực quan hóa dữ liệu 22KHDL
- Lab: Lab 03 - Trực quan hóa dữ liệu với Power BI
- Giảng viên: Võ Nhật Tân

## 1. Mục tiêu bài lab

Nhóm cần sử dụng **Power BI** để xây dựng một bài phân tích dữ liệu hoàn chỉnh từ một **relational dataset**. Kết quả cuối cùng phải thể hiện được:

- Khả năng tìm hiểu và sử dụng các chức năng cơ bản của Power BI
- Khả năng xây dựng mô hình dữ liệu từ nhiều bảng có liên kết
- Khả năng đặt câu hỏi phân tích và trực quan hóa dữ liệu phù hợp
- Khả năng thiết kế dashboard rõ ràng, có tính tương tác
- Khả năng rút ra insight và kết luận từ dữ liệu

## 2. Quy định chung

- Bài làm theo nhóm từ **3 đến 5 thành viên**
- Thành viên không tham gia sẽ **không có điểm**
- Bài làm phải tuân thủ đúng yêu cầu đồ án
- Nếu có tài liệu tham khảo, phải ghi đầy đủ trong báo cáo
- Có thể dùng các công cụ như ChatGPT, GitHub Copilot... để tham khảo, nhưng nội dung phải được kiểm tra và chỉnh sửa phù hợp
- Nếu lạm dụng AI hoặc nội dung sai lệch, không phù hợp, nhóm có thể bị trừ tối đa **50% số điểm**
- Bài giống nhau sẽ bị **0 điểm**

## 3. Yêu cầu bắt buộc

### 3.1. Công cụ sử dụng

- Chỉ sử dụng **Power BI** để thực hiện trực quan hóa và xây dựng dashboard
- Không sử dụng các sample dataset có sẵn trong Power BI cho bài chính

### 3.2. Yêu cầu về dataset

Dataset được chọn phải thỏa các điều kiện sau:

- Là **relational dataset**
- Có ít nhất **2 đến 3 bảng dữ liệu**
- Các bảng phải có **relationship rõ ràng**
- Dữ liệu phải đủ lớn để có thể phân tích và trực quan hóa

Nguồn dữ liệu có thể lấy từ:

- Kaggle
- World Bank
- Data.gov
- Các nguồn open dataset khác

### 3.3. Yêu cầu về data model

Nhóm phải xây dựng mô hình dữ liệu trong Power BI và mô tả rõ:

- Danh sách các bảng dữ liệu được sử dụng
- Ý nghĩa của từng bảng
- Các trường dữ liệu quan trọng
- Các khóa liên kết giữa các bảng
- Các kiểu quan hệ như:
  - One-to-one
  - One-to-many
  - Many-to-one

## 4. Nhiệm vụ cần thực hiện

### 4.1. Tìm hiểu Power BI

Nhóm cần trình bày:

- Tổng quan về Power BI
- Các thành phần chính:
  - Power BI Desktop
  - Power BI Service
  - Power BI Report
- Các chức năng cơ bản:
  - Import dữ liệu
  - Data transformation bằng Power Query
  - Data modeling
  - Visualization
  - Dashboard

Lưu ý:

- Phần minh họa cho các chức năng trên phải dùng **dataset mẫu hoặc dataset đơn giản**
- Không dùng dataset chính của bài toán để minh họa phần này

### 4.2. Xác định mục tiêu phân tích

Nhóm cần:

- Xác định một chủ đề phân tích chung
- Đặt ra các câu hỏi phân tích cụ thể dựa trên dataset đã chọn
- Số lượng câu hỏi tối thiểu là:

`Số thành viên x 3 câu hỏi`

Ví dụ:

- Nhóm 3 người: ít nhất 9 câu hỏi
- Nhóm 4 người: ít nhất 12 câu hỏi
- Nhóm 5 người: ít nhất 15 câu hỏi

### 4.3. Phân tích dữ liệu và trực quan hóa

Nhóm cần dùng Power BI để xây dựng các biểu đồ trả lời các câu hỏi phân tích đã đề ra.

Các loại biểu đồ nên sử dụng gồm:

- Bar chart
- Line chart
- Pie chart
- Scatter plot
- Map
- Table hoặc Matrix
- Các biểu đồ khác nếu phù hợp

Yêu cầu:

- Chọn loại biểu đồ phù hợp với bản chất dữ liệu
- Sử dụng màu sắc hợp lý và giải thích ý nghĩa lựa chọn màu sắc nếu cần
- Sau mỗi biểu đồ hoặc nhóm biểu đồ cần có phần nhận xét, phân tích, và kết luận ngắn

### 4.4. Xây dựng dashboard

Nhóm phải thiết kế một hoặc nhiều dashboard tổng hợp kết quả phân tích.

Dashboard cần:

- Có bố cục hợp lý
- Dễ theo dõi
- Thể hiện rõ insight chính
- Có filter hoặc slicer để tương tác dữ liệu
- Có luồng trình bày như một storyboard phân tích

### 4.5. Nhận xét và kết luận

Cuối bài cần:

- Tổng hợp các kết quả phân tích quan trọng
- Nêu các xu hướng nổi bật
- Rút ra insight có ý nghĩa
- Kết luận cho bài toán phân tích chung của nhóm

## 5. Báo cáo cần có những gì

Báo cáo phải ngắn gọn nhưng đầy đủ, tối đa **24 trang**, không tính:

- Trang bìa
- Mục lục
- Tài liệu tham khảo

### Nội dung báo cáo

#### 1. Thông tin nhóm

Bảng thông tin gồm các cột:

- STT
- MSSV
- Họ và tên
- Tỷ lệ đóng góp của từng thành viên

#### 2. Tìm hiểu công cụ Power BI

- Giới thiệu tổng quan Power BI
- Trình bày Power BI Desktop, Power BI Service, Power BI Report
- Trình bày các chức năng cơ bản:
  - Import dữ liệu
  - Data transformation
  - Data modeling
  - Visualization
  - Dashboard
- Minh họa bằng dataset mẫu hoặc dataset đơn giản

#### 3. Giới thiệu bài toán phân tích

- Tổng quan bài toán phân tích dữ liệu
- Mục tiêu phân tích chung của nhóm
- Mục tiêu phân tích cụ thể của từng thành viên

#### 4. Dataset và quy trình xử lý dữ liệu

- Nguồn dữ liệu và lý do chọn dataset
- Mô tả các bảng dữ liệu
- Mô tả các trường dữ liệu quan trọng
- Mô tả các relationship giữa các bảng
- Trình bày quy trình thu thập, làm sạch, biến đổi dữ liệu

#### 5. Phân tích dữ liệu và trực quan hóa

Với từng thành viên hoặc từng phần phân tích, cần trình bày:

- Mục tiêu phân tích
- Biểu đồ hoặc phương pháp trực quan hóa được sử dụng
- Kết quả phân tích
- Nhận xét và kết luận

Ngoài ra:

- Cần chèn hình ảnh minh họa từ Power BI
- Cuối phần này phải tổng hợp các insight chính của toàn bài

#### 6. Tài liệu tham khảo

Liệt kê đầy đủ:

- Nguồn dữ liệu
- Tài liệu đọc thêm
- Website tham khảo
